title: テストエンジニアとの協業
---

このアプリの開発において、テストエンジニアとの協業により得られる利点を検証しました。
以下に、その検証により得られた知見や課題を紹介します。

## テストエンジニアと協業する理由

モバイルアプリのテストは、デバイスの多様性、モバイル環境の制約、プラットフォームの継続的な変化などにより、Webアプリよりも複雑で時間を要する場合があります。
このような課題に対して、テストエンジニアとの協業が1つの解決策として挙げられます。

テストエンジニアと協業する利点は次のとおりです。

1. **品質の向上**：テストエンジニアが開発者と異なる観点でテストを実施することにより、アプリの品質向上が期待できます。
2. **問題の早期発見**：テストエンジニアは開発と並行してテストを実施できるので、問題の早期発見が期待できます。
3. **多様なデバイスの準備が不要**：テストベンダーは多様なデバイスを用意できるため、プロジェクト毎のデバイス購入が必要ありません。
4. **継続的な改善**：モバイルプラットフォームは継続的に変化します。新しいOSバージョン、API、およびハードウェアへの対応が必要です。テストエンジニアが継続的なテストを実施することにより、アプリ品質の維持・改善が期待できます。また、アプリ基盤や使用ライブラリのアップデートが行われた際にも、テストエンジニアがテストを実施し、問題を発見できます。
5. **チームの生産性向上**：開発と並行してテストエンジニアがテストを実施できます。これにより、開発者は開発に集中でき、チーム全体の生産性向上が期待できます。

## テストエンジニアとの協業で得た課題

このアプリの開発において、テストエンジニアと協業した結果、以下の課題が浮き彫りとなりました。

- テストエンジニアは開発者とは異なり、ソースコードを読み解くスキルに長けているとは限りません。そのため、ホワイトボックステストにおけるソースコードの網羅性確認は、開発者に委ねられます。
- テストエンジニアは、ブラックボックステストを中心とした作業が多いです。要件定義や設計工程での作業比率が少ないため、状況によっては作業が空いてしまうことがあります。
- テストエンジニアは、作業マニュアルの作成は可能ですが、設計書の作成は難しいです。
- テストエンジニアのスキルによっては、テストデータの準備に時間を要するため、開発者側のタスクとなる場合があります。
- テストエンジニアは、テスト仕様書を作成するために開発者より仕様の説明を受ける必要があります。この説明には十分な時間を割く必要があり、生産性に悪影響を及ぼす可能性があります。

これらの課題を把握して計画することで、テストエンジニアとの協業において生産性が期待できると考えます。
