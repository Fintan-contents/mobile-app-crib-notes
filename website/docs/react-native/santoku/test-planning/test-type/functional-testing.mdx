## 目的

- クラス、関数に実装された各業務ロジックについて、仕様との差異を発見する
- APIにより取得した結果について、仕様との差異を発見する
- 画面ごとの実行結果について、仕様との差異を発見する
- アプリの利用シナリオをベースとした実行結果について、仕様との差異を発見する

## テスト方法

上記観点のテストは以下のとおり、3種類の方法で実施します。テスト観点ごとに、どのテスト方法でテストするかを記載してあります。

- (A) 自動テスト
- (B) エミュレータ／シミュレータもしくは実機で手動テスト
- (C) 実機で手動テスト
