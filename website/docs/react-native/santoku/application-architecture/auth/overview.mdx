title: 認証方式

アプリを安全に利用するためには、ユーザが本人であることを確認する必要があります。このアプリでは、ユーザを確認する手段としてID・パスワードを用いた認証機能を提供します。

また、モバイル端末の紛失や盗難リスクに対応するため、PINコードやユーザの生体属性を使用して、ユーザが端末の所有者であることを確認する機能も提供します。

## 認証機能

このアプリで提供する機能は以下になります。

| 機能 | 概要 |
|:--|:--|
| サインアップ | このアプリでは、ユーザはサインアップ操作なしにアプリを利用できます。アカウントIDとパスワードは自動生成してアカウントを作成します。作成したアカウントIDとパスワードは、ログイン資格情報として端末内の安全な場所（以降、セキュアストレージと呼びます）に保存します。<br/>このアプリでは、ユーザは複数のアカウントを作成できます。|
| ログイン | アカウントIDとパスワードを用いて、Cookieベースの認証処理を実施します。認証処理後に払い出されたセッションIDをCookieに設定することで認証状態を維持します。<br/>ログインは、ログイン画面でアカウントIDを入力する方法と、端末に保存されたログイン資格情報を用いて自動ログインする2つの手段を提供します。
| ログアウト | ログイン処理後に払い出されたセッションIDを破棄します。|
| 端末認証 | PINコード（またはパスコード）や、指紋・顔・虹彩を用いて端末の所有者であることを確認します。 |
| ログイン資格情報の管理 | ログイン資格情報を、セキュアストレージで管理します。 |
