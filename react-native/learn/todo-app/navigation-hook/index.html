ナビゲーションの型定義

useNavigationのカスタムフック#

useNavigationの型定義にあるとおり、useNavigationに"正しい"型を指定することで、型安全にナビゲーション操作できます。 しかしながら、useNavigationの型定義は複雑です。 "正しい"型を指定すれば型安全ですが、各画面に型指定したuseNavigationフックを宣言した場合、それらをメンテナンスし続けるのは大変（かつ間違っても気づきにくい）です。 "正しい"型を維持しつつ管理を楽にするため、各画面で宣言するのは避け、ナビゲータ毎のカスタムフックを事前に用意するほうがいいでしょう。

ここでは、AuthedStackNav、UnauthedStackNavそれぞれのカスタムフックを用意します。

まずはAuthedStackNavのカスタムフックを用意します。 /src/navigation/hooks/AuthedStackHooks.tsを追加してください。

/src/navigation/hooks/AuthedStackHooks.ts
import type {CompositeNavigationProp, RouteProp} from '@react-navigation/native';
import {useNavigation, useRoute} from '@react-navigation/native';
import {StackNavigationProp} from '@react-navigation/stack';
import {AuthedStackParamList, RootStackParamList} from 'navigation/types';
type AuthedStackParamListKeys = keyof AuthedStackParamList;
type AuthStackNavigationProp<T extends AuthedStackParamListKeys> = CompositeNavigationProp<
StackNavigationProp<AuthedStackParamList, T>,
StackNavigationProp<RootStackParamList>
>;
export const useAuthedStackNavigation = <T extends AuthedStackParamListKeys>() =>
useNavigation<AuthStackNavigationProp<T>>();
export const useAuthedRootStackRoute = <T extends AuthedStackParamListKeys>() =>
useRoute<RouteProp<AuthedStackParamList, T>>();

上記コードではkeyof演算子を使用しています。 ここでkeyof演算子について簡単に説明します。 keyof演算子の使用例として次のコードを確認してください。

interface Person {
name: string;
age: number;
location: string;
}
type K1 = keyof Person; // "name" | "age" | "location"

上記コードから分かるとおり、keyof演算子を使うとオブジェクトのすべてのキーを、文字列リテラル型の合併として取得できます。 AuthedStackParamListKeysはAuthedStackParamListのキーを文字列リテラル型の合併として取得しているため、次のコードと同等です。

type AuthedStackParamListKeys = "Welcome" | "Login" | "Instructions";

また、AuthedStackHooks.tsの中でuseRouteのカスタムフックも定義しています。 useRouteはrouteオブジェクトへのアクセスを提供するフックです。 今回のToDoアプリでは使用しませんが、useRouteのカスタムフックも合わせて準備しておくといいでしょう。

では、同様にUnauthedStackNavのカスタムフックを用意します。 /src/navigation/hooks/UnauthedStackHooks.tsを追加してください。

/src/navigation/hooks/UnauthedStackHooks.ts
import type {CompositeNavigationProp, RouteProp} from '@react-navigation/native';
import {useNavigation, useRoute} from '@react-navigation/native';
import {StackNavigationProp} from '@react-navigation/stack';
import {RootStackParamList, UnauthedStackParamList} from 'navigation/types';
type UnauthedStackParamListKeys = keyof UnauthedStackParamList;
type UnauthedStackNavigationProp<T extends UnauthedStackParamListKeys> = CompositeNavigationProp<
StackNavigationProp<UnauthedStackParamList, T>,
StackNavigationProp<RootStackParamList>
>;
export const useUnauthedStackNavigation = <T extends UnauthedStackParamListKeys>() =>
useNavigation<UnauthedStackNavigationProp<T>>();
export const useUnauthedStackStackRoute = <T extends UnauthedStackParamListKeys>() =>
useRoute<RouteProp<UnauthedStackParamList, T>>();

カスタムフックをnavigation/hooksとして再エクスポートします。 /src/navigation/hooks/index.tsを追加してください。

/src/navigation/hooks/index.ts
export * from './AuthedStackHooks';
export * from './UnauthedStackHooks';

note

UnauthedStackNavのuseUnauthedStackNavigationは特定画面から使われるわけではないので、画面名が指定していません。 このように、画面名を省略してカスタムフックを使用することも出来ます。

修正できたら実行して、動作に問題がないかを確認してください。
