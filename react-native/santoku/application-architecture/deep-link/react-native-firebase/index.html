React Native Firebaseの課題対応

React Native Fireabaseで発生する2つの課題に対応します。

  • iOSで、Firebase Dynamic LinksのmatchTypeを検証できない
  • iOSで、アプリ起動の時間が短いとディープリンクURLを取得できない

他のユーザが開いたディープリンクURLを読み取る可能性がある#

Firebase Dynamic Linksは、iOSで初回起動時にディープリンクURLを取得するためにクリップボードを利用するとドキュメントに記載されています。ただし、実際はクリップボードの値だけでなく、デバイス情報なども利用しています。

このため、例えばクリップボードが空だったとしてもディープリンクURLを取得できる場合があります。こういった場合は、ユーザ自身が実際に開こうとしたわけではないディープリンクURLが取得されている可能性があります。

ディープリンクURLをユーザが開いたものかどうかを判定するためにmatchTypeという項目があります。

このmatchTypeの値がuniqueでない場合は誤って受け取ったディープリンクURLである可能性があるため、アプリケーションでこのURLは利用しません。

ただし、React Native Firebase Dynamic Linksでは、matchTypeをJavaScript Moduleに渡していないため、アプリケーションが参照できません。 そのためReact Native Firebase Dynamic Linksにpatch-packageでパッチを適用し、判定できるようにします。

note

Androidの場合はドキュメントのフローにもあるように、Google Play Servicesを経由してIntentでURLを渡すため、デバイス外に情報を持つことはありません。

アプリ起動の時間が短い場合にディープリンクURLを取得できない#

iOSアプリではアプリ起動の時間が短いとディープリンクURLを取得できないissueがあります。このissueに対応するため、getInitialLinkではなく、React Native LinkingのgetInitialURLを利用します。

React Native Linkingでは、コンテンツリンクURLではなく、ディープリンクURLが取得されるためresolveLinkを利用してコンテンツリンクURLを取得します。

note

AndroidアプリではReact Native Linkingにissueがあるため、iOSのみReact Native Linkingを利用します。
