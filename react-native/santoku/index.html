React Nativeのサンプルアプリケーション

モバイルアプリの開発は短期間での開発が要求される傾向にあります。そのため、技術選定や方式設計など、開発準備を十分に行う時間を確保できないことが多いと感じています。

そこで、React Nativeアプリの開発をスムーズに進行できるよう、アプリ開発現場で役立つドキュメント類および、ソースコードを公開しています。これらは時間割共有アプリを題材として、実際の開発プロセスに沿った形で作成しています。

attention

サンプルアプリケーションの開発手法はウォーターフォールモデルを採用しています。その理由は以下のとおりです。

  • 主要な読者として想定しているウォーターフォールモデルに慣れている開発者にとっては、公開するサンプルがウォーターフォールモデルで作成する成果物と一致していた方が理解しやすい
  • ウォーターフォールモデルに沿って各工程ごとの成果物を一通り作成し、その後に各成果物の効果を振り返ることで、モバイルアプリの開発を成功させるために最低限必要な成果物を見極めたい

Requirements »

このアプリの機能要件と非機能要件を定義しています。特にモバイルアプリの非機能要件を重点的に扱っています。

Application Architecture »

このアプリの方式設計です。モバイルアプリを開発したときに、初めにきちんと検討するべきだったとふりかえったテーマから順にまとめています。

Test Planning »

このアプリのテスト計画です。モバイルアプリ特有の観点に重点を置いて、テストの目的やプロセスを定義しています。

Design »

このアプリの外部設計です。画面遷移図や画面仕様、API仕様などの外部設計工程で必要なドキュメント類をまとめています。

Development »

このアプリの開発ガイドです。このアプリの開発者を対象として、各種ルールや開発ノウハウについてまとめています。

Glossary »

このアプリの用語集です。このアプリで使用する用語と意味の統一を目的としています。

Decision Record »

このアプリのアーキテクチャや機能設計に関する判断の記録です。実際に方式を設計する際に、このサンプルアプリケーションをより有効に活用できるよう、設計した方式だけでなく検討過程も記録しています。
