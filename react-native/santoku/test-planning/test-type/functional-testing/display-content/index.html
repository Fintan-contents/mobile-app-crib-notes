表示内容に関する観点

テキストや項目、表示するメッセージのテスト観点をまとめています。 それぞれのテスト観点毎の検証方法は、テスト方法にまとめています。

テキスト表示​

NO詳細観点補足テスト方法
1固定文字列仕様通りの文章が表示されていること-B
2外部データ仕様通りの文章が表示されていること（別の外部データが誤って表示されていないこと）-B
3外部データ外部データに含まれる改行と同じ位置で画面上でも改行されて表示されていること-B
4外部データ外部データをマスキングして表示する設計の場合、マスキングされた値が表示されていること-B
5外部データサロゲートペア文字を許容する場合、サロゲートペア文字が表示されていること例：絵文字などを表示するB
6フォーマット仕様通りのフォーマットで表示されていること-B

一覧表示項目​

NO詳細観点補足テスト方法
1リスト表示デフォルトのソート順が設計で定められている場合、仕様通りのソート順で表示されていること-B
2リスト表示表示データの取得に失敗した場合、仕様通りの内容が表示されること-B

入力項目​

NO詳細観点補足テスト方法
1テキスト入力パスワード等の機密情報の入力欄は、入力内容がアスタリスクなどの記号に置き換えた形で表示されていること-B
2テキスト入力現在の入力文字数を表示する場合、サロゲートペア文字や異体字を含む文字列を入力しても正しい入力文字数が表示されること-B
3テキスト入力プレースホルダーに表示されている内容が仕様通りに表示されていること-B
4セレクトボックスデフォルトの選択状態が仕様通りであること-B
5セレクトボックス表示される選択肢が仕様通りの選択肢のみであること-B
6セレクトボックス表示される選択肢が動的に変化する場合、変化する条件に応じて仕様通りの選択肢のみが表示されていること-B
7ラジオボタンデフォルトの選択状態が仕様通りであること-B
8ラジオボタン表示される選択肢が動的に変化する場合、変化する条件に応じて仕様通りの選択肢のみが表示されていること-B
9チェックボックスデフォルトのチェック状態が仕様通りであること-B
10チェックボックス表示される選択肢が動的に変化する場合、変化する条件に応じて仕様通りの選択肢のみが表示されていること-B

UI部品​

NO詳細観点補足テスト方法
1データUI部品で表示するデフォルト値が仕様通りであること-B
2データUI部品で表示する項目の取得元が仕様通りであること-B
3データUI部品で表示する項目の編集結果が仕様通りであること例：想定通りの計算結果であることB
4データUI部品で表示する項目のソート順が仕様通りであること例：プルダウンやラジオボタンなどのUI部品B
