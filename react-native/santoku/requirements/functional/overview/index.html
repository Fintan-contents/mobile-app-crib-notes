機能要件

サンプルアプリケーションの機能要件についてまとめたものです。 サンプルアプリケーションの主要機能として、「アカウント機能」「チーム機能」「時間割機能」があります。それぞれの機能要件を次に示します。

✅ が付いている項目はこのアプリの機能として公開されているものになります。

アカウント​

  • ユーザは、アカウントを作成することでアプリを利用できる
  • ✅ ユーザは、有効な利用規約に同意した場合のみアプリを利用できる
  • ユーザは、ログイン中のアカウントからログアウトできる
  • ユーザは、過去に作成したアカウントでログインできる
  • アカウントは、自身のアバター（プロフィール画像）を設定できる
  • アカウントは、フォトライブラリの画像をアバターとして設定できる
  • アカウントは、撮影した画像をアバターとして設定できる
  • アカウントは、他アカウントのアバターを参照できる

アカウントの保護​

  • ユーザは、パスワードを記憶する必要なく安全にアカウントを利用できる
  • ユーザは、ログインしていない場合はアカウント作成機能とログイン機能のみ利用できる
  • ユーザは、ログインしている場合はログイン中のアカウントの権限で有効な機能を全て利用できる

チーム​

  • アカウントは、チーム名を指定してチームを作成できる
  • チームの作成アカウントは、作成したチームのチームメンバーとなる
  • チームの作成アカウントは、作成したチームのオーナー権限を持つ
  • アカウントは、自分が参加しているチームのチームメンバー一覧を参照できる

チームメンバー​

  • アカウントは、一定時間のみ有効な招待コードを入力することでチームに参加できる
  • アカウントは、一定時間のみ有効なQRコードの読み取りでチームに参加できる
  • アプリは、アカウントが最後に参加したチームをデフォルトのチームに設定する（デフォルトチームの選択機能を追加したら、この機能は廃止する）
  • チームメンバーは、他のアカウントをチームに招待できる

時間割テンプレート​

  • チームメンバーは、時間割テンプレートをチームに登録できる
  • チームメンバーは、時間割テンプレートに時間割テンプレート名と複数のコマを設定できる
  • チームメンバーは、時間割テンプレートのコマにコマ名と開始・終了時刻を設定できる
  • チームメンバーは、チームの時間割テンプレートを編集できる
  • チームメンバーは、時間割テンプレートのコマを編集できる
  • チームメンバーは、時間割テンプレートのコマを削除できる

当日の時間割​

  • チームメンバーは、本日以降の当日の時間割をテンプレートから選択できる
  • アプリは、当日の時間割を一日1チームあたり1つに制限する
  • チームメンバーは、当日の時間割を開始できる
  • アプリは、当日の時間割が開始されると、時間割を開始したチームメンバー以外に時間割開始を通知する
  • チームメンバーは、チームの当日の時間割の時間割テンプレート名とすべてのコマを参照できる
  • チームメンバーは、チームで現在進行中のコマを知ることができる

アプリの運用​

  • アプリ管理者は、ユーザにアプリのアップデートを強制できる
